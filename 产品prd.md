. 产品概述
产品一句话：让用户通过“成就感经历”反推稳定优势与偏好，生成30个职业岗位推荐与职业路径可能性分析，帮助用户更早做对选择，而不是在投递层面反复内耗。
1. 背景与问题陈述
求职/职业规划的常见痛点在于：
职业方向不清：用户知道自己“想做什么”（或跟风热门），但不知道自己“为什么适合”（证据链缺失）。
越努力越焦虑：投入大量时间学习与投递，但没有形成“长期可持续”的路线判断。
工具是 Job Planner，不是 Career Developer：多数产品做“岗位清单/简历优化/投递策略”，但没有回答关键问题：
“我是什么样的人？我不该做什么？我适合怎样的成长路径？”
你的核心洞察是：选择比努力重要；通过“减法”做反向筛选，才能让人又有钱又开心。
2. 目标与非目标
2.1 产品目标（Goals）
G1 可解释的匹配：每个岗位推荐必须能追溯到用户的具体经历细节（事实 vs 推断分区）。
G2 Career Path 分析：不仅给岗位，还给“未来怎么走”的可能路径（1年/3年/5年）。
G3 30个岗位输出有层级：按“现在可试 / 可迁移 / 探索潜力”分层呈现。
G4 反向筛选：明确说明“不适配的原因”（环境压力/节奏/人际/工作内容类型等）。
G5 用户可控可编辑：用户可对“能力归因/偏好判断”进行确认、修改，并影响后续推荐。
2.2 非目标（Non-Goals，v1不做）
不保证结果：不承诺拿 offer、涨薪、改变命运。
不做全量实时岗位市场抓取（可用静态/半静态岗位模型即可）。
不做心理治疗/强人格结论：避免“测出你是谁”的绝对化表达。
3. 用户与场景
3.1 核心用户
在校生/应届生：经历碎片、容易跟风选赛道。
转行/摇摆者（0-3年）：有经历但不知道“哪个方向是可持续的”。
稳定偏好强的人：不适合高强度大厂竞争节奏，容易误判。
3.2 典型场景（User Stories）
我把几段让我有成就感的经历输入后，希望系统告诉我：我稳定擅长的能力簇是什么，以及我可能适合哪些职业方向。
我看到一堆岗位都“看起来不错”，但想知道哪些更可能让我舒服且成长，并明确哪些我不该去硬撑。
我选了某条路径后，希望系统给我一个最小行动计划：30天能做什么来验证方向对不对。
4. 价值主张与差异化
4.1 核心价值
从成就反推“稳定优势”：把“做得好/做得开心”的部分结构化出来。
从能力映射到岗位：形成“经历 -> 能力 -> 岗位 -> 路径”的证据链。
从岗位到路径：回答“未来怎么成长”，帮助做长期决策。
强调减法：用“不适配风险”减少错误投入。
4.2 相比 Job Planner 的差异
Job Planner 更像“怎么找”，你更像“怎么选”。
不止推荐，而是解释 + 反向筛选 + 路径分析。
5. 核心概念（你产品的方法论）
成就驱动的职业基因（Career DNA） =
能力簇（做成了什么）
驱动力（为什么愿意做）
情境偏好（在什么环境/节奏下最好发挥）
价值观与边界（我不喜欢什么/我不能长期承受什么）
最终形成一个“职业适配模型”，用它来生成 30 个岗位与路径可能性。
6. 功能范围总览
6.1 MVP（v1 必须交付）
成就经历采集
引导式输入：至少 3 段“有成就感”的经历（可为项目/实习/社团/学习/兼职等）
每段经历包含：情境、你做了什么、如何做、结果/影响、当时的感受（尤其是“让你上头/满足/安心”的点）
职业画像输出（结构化）
能力簇（Top 5-8）
驱动力/偏好（可用标签+解释）
边界/不适配因素（Top 3-5）
30个岗位推荐（分层）
第一梯队：现在就能尝试（匹配强）
第二梯队：可迁移方向（需要补齐1-2个关键能力）
第三梯队：探索潜力（长期可能）
职业路径可能性分析
每个方向给：未来成长路径（1/3/5年）、需要补的能力、风险点、不适配人群画像
30天验证行动计划
每个方向给 3-5 个“验证任务”（例如：做一个小项目、补一项能力、产出一个作品、找一个对标角色对话等）
用户确认/编辑机制
用户可以确认/修改：能力归因、偏好判断、每个方向的“不适合原因”是否准确
后续推荐基于用户反馈迭代
6.2 v1.1 / v2（后续）
多轮对话式追问（把不确定性变小）
学习/作品生成辅助（只给方向与任务，不强绑定课程）
对接岗位 JD/自测题（验证能力是否真的可迁移）
形成“职业档案”（可长期复用）
7. AI 工作流（建议）
经历解析与证据抽取
从每段经历中抽取“事实要点”（动作、结果、规模、协作等）
能力归因与驱动力识别
输出能力簇 + 一句话“为什么你能/你愿意”
岗位库映射
将能力簇映射到岗位能力模型（岗位标签体系）
生成30个岗位并分层排序
分层依据：匹配度 + 可迁移成本 + 长期可持续性
路径分析与反向筛选
给每个方向：成长路径、补齐项、不适配原因（来自偏好/边界）
行动计划生成
将“验证任务”落到可执行、可量化的最小行动
关键原则：不编造经历细节；若需要假设，必须标注为“示例/待验证”。
8. 用户关键路径（Flows）
首次使用
创建职业档案 -> 输入成就经历（3-8段） -> 查看职业画像 -> 查看30岗位分层推荐 -> 选择 1-3 条路径 -> 查看路径分析 -> 获取30天行动计划 -> 保存导出
纠错与二次迭代
用户修改“能力归因/偏好标签” -> 系统更新路径可能性 -> 重新生成行动计划
